# Release Versioning

HUMINEX architecture docs use semantic versioning:

- `MAJOR`: breaking documentation structure/spec changes
- `MINOR`: new sections/modules/diagrams
- `PATCH`: incremental updates and corrections

The rendered portal footer shows:

- `Version: major.minor.patch`
- `Released: UTC timestamp`

Source file:

- `huminex-architecture/version.json`
